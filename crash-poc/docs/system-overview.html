AI Crash Analysis System

Automated crash detection, AI-powered root cause analysis, and self-healing code fixes for Dell product applications.

.NET 9.0 React + Vite Google Gemini AI GitHub API WER Integration

System Overview

This system provides an end-to-end pipeline from crash detection to automated code repair. It collects crashes from Dell product applications via the Windows Error Reporting (WER) API, stores and categorizes them using PDB-based stack symbolication, analyzes root causes with Google Gemini AI, automatically generates code fixes, and opens GitHub Pull Requests for developer review.

💣

Crash Collection

Collects crash dumps via WER API or local Dell app crash generation. Extracts minidump metadata including exception codes, stack traces, and module info.

📊

Smart Bucketing

Groups crashes by normalized stack signatures using PDB-resolved symbols. SHA-256 hashed bucket IDs ensure deterministic grouping across runs.

🤖

AI Analysis

Google Gemini analyzes crash context — stack traces, source code, and PDB info — to identify root causes with confidence scoring.

🛠

Auto-Fix & PR

Generates code fixes, creates branches, commits changes, and opens GitHub PRs automatically. Developers review AI-generated patches.

Architecture

Data Flow Pipeline

1

Crash Generation

DellDigitalDelivery.App simulates a Dell product with intentional bugs (buffer overrun, null ref, stack overflow, heap corruption, I/O error). Each run generates 20+ minidump files with metadata JSON.

2

WER API Collection

WerApiClient scans the local crash-dumps directory, reads metadata.json files, and creates CrashReport objects matching the WER API response format. Falls back to synthetic data if no local dumps exist.

3

Dump Storage & Metadata Extraction

LocalDumpStore copies dump files with SHA-256 validation. DumpMetadataExtractor reads MDMP headers and companion metadata.json to extract exception codes, stack frames, thread counts, and faulting modules.

4

Crash Bucketing

Bucketizer normalizes stack traces, extracts top 3 key application frames, and computes SHA-256 bucket IDs. Crashes with identical signatures land in the same bucket.

5

AI Analysis (Gemini)

CrashAnalyzer sends crash context (stack trace + PDB-matched source code) to Google Gemini. The AI identifies root causes, affected files/functions, confidence scores, and generates complete code fixes.

6

Auto-Fix & PR Creation

AutoFixWorkflow orchestrates the full cycle: AI analysis → code generation → GitHub branch creation → file commit → Pull Request. Results are stored in the AIFixes database table.

7

Developer Dashboard

React-based dashboard with login, crash overview, bucket details, AI analysis results, confidence scores, suggested fixes, and PR links. Developers review and merge AI-generated patches.

CrashCollector.Console .NET 9.0

7-step pipeline orchestrator that processes crashes end-to-end:

  • WerApiClient — WER REST API with OAuth 2.0 + local dump fallback
  • LocalDumpStore — File storage with SHA-256 integrity + local:// URL support
  • DumpMetadataExtractor — WinDbg, MDMP header parsing, and DellApp metadata loading
  • Bucketizer — Stack-based crash grouping with normalized signatures
  • LocalSymbolServer — PDB discovery and symbol publishing

CrashCollector.AI AI

  • GeminiClient — Google Gemini REST API client with token tracking
  • CrashAnalyzer — Prompt engineering with crash context + source code matching
  • GitHubClient — Branch creation, file commits, and PR automation
  • AutoFixWorkflow — Full orchestration: analyze → fix → branch → PR

Database Schema

SQLite database at data/crashes.db with 5 tables:

Crashes (CrashId PK, AppName, AppVersion, Timestamp, ExceptionCode,
         ExceptionName, FaultingModule, ThreadCount, DumpPath, DumpSizeMB,
         DumpSha256, FailureBucket, ExtractionMethod, CreatedAtUtc)

Buckets (BucketId PK, ExceptionCode, KeyFrame1..3, CrashCount,
         FirstSeenUtc, LastSeenUtc)

CrashBucketMap (CrashId FK, BucketId FK, MappedAtUtc)

AIAnalysis (AnalysisId PK, BucketId FK, RootCause, Confidence,
            SuggestedFix, AffectedFile, AffectedFunction, Status,
            ErrorMessage, PromptTokens, ResponseTokens,
            CreatedAtUtc, CompletedAtUtc)

AIFixes (FixId PK, AnalysisId FK, BucketId FK, BranchName,
         CommitSha, PRNumber, PRUrl, PRTitle, PRStatus,
         FixDescription, FilesChanged, ErrorMessage,
         CreatedAtUtc, UpdatedAtUtc)

Quick Start

# 1. Generate crash dumps from the sample Dell app
dotnet run --project DellDigitalDelivery.App

# 2. Run the crash collection pipeline
dotnet run --project CrashCollector.Console

# 3. Start the API server
dotnet run --project CrashCollector.Api --urls http://localhost:5100

# 4. Start the React dashboard (in another terminal)
cd crash-dashboard && npm install && npm run dev

# 5. Open http://localhost:3000 and sign in (any credentials work)
